# Project Safety Skill

When working in this project:

1. Check the current Git status before making file changes.
2. Explain the intended change before editing files.
3. Keep edits small and easy to review.
4. Do not push to GitHub unless explicitly instructed.
5. After changes, summarize what changed and recommend the next verification step.
